This package contains the essential functionality for graph transformation. It relies mostly on groove.graph, implementing graph functionality, and groove.match implementing graph matching.

An important part of the contents of the package is taken up by the interfaces GraphTest, with sub-interfaces GraphCondition and GraphPredicate. As the names imply, these provide ways to formulate and test properties of graphs. Such properties are constructed as trees (or forests) of graphs connected by morphisms, essentially as set out in [Rensink, ICGT 2004].

The interface Rule builds on GraphCondition and adds to it the functionality of actually transforming graphs. This is based on the algebraic approach: a rule has a left hand side and a right hand side, both of which are graphs, with a morphism between them that captures the changes embodies by the rule. To apply a rule it is matched to a graph, which results in a RuleEvent capturing an instance of the rule; the combination of host graph and rule event then leads to a RuleApplication from which the result (target) graph can be obtained.

Rules are combined into RuleSystems, which (provided with a start graph) in turn give rise to GraphGrammars.
